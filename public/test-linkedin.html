LinkedIn Share Test Page

⚠️ IMPORTANT: Localhost Limitation

LinkedIn cannot access localhost URLs! This is why you're seeing "Cannot display preview" in the LinkedIn share dialog.

You must deploy your app to a public URL for LinkedIn to properly scrape the meta tags and display the image preview.

Quick Solutions

Option 1: Use ngrok (Immediate Testing)

# Install ngrok globally
npm install -g ngrok

# Start your Next.js app
npm run dev

# In another terminal, expose localhost:3000
ngrok http 3000

This will give you a public URL like https://abc123.ngrok.io that LinkedIn can access.

Option 2: Deploy to Vercel (Recommended)

# Install Vercel CLI
npm install -g vercel

# Deploy your app
vercel

This will give you a public URL like https://your-app.vercel.app

Option 3: Netlify Drop

Go to netlify.com/drop and drag your project folder for instant deployment.

After Deploying to Public URL

Once you have a public URL (from ngrok, Vercel, or Netlify), replace localhost:3000 with your public URL in the LinkedIn share component.

// In LinkedInShareFeature.tsx
const sharePageUrl = "https://your-public-url.com/share";

Meta Tags Verification

These are the meta tags that LinkedIn will read (once it can access your public URL):

<meta property="og:title" content="XperThrivr - Empowering Individuals">
<meta property="og:description" content="Get valuable insights and career advice from experienced professionals. Book a session and grow together!">
<meta property="og:image" content="/api/og">
<meta property="og:image:width" content="1200">
<meta property="og:image:height" content="630">
<meta property="og:image:type" content="image/png">

Testing Steps

  1. Deploy to a public URL using one of the methods above
  2. Update your share component with the public URL
  3. Test the share page at your public URL
  4. Use LinkedIn Post Inspector to verify meta tags
  5. Test sharing - image should now appear in full size!
